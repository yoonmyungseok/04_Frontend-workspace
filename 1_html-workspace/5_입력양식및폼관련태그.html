폼 관련 태그

입력 양식의 틀을 의미
form 태그 내에 있는 submit 버튼을 클릭 시 해당 form 내에 작성된 사용자가 입력한 값들을 서버의 컨트롤러로 넘기면서 요청하는 역할을 수행함
form action="보낼경로(목적지)" method="보내는방법"
*action 속성: 해당 from 내에 사용자가 작성한 값들을 전달하면서 요청할 서버의 경로를 제시(어느 기능을 담당하는 컨트롤러로 보낼건지 지정)
*method 속성: 요청 전송 방식을 지정하는 속성( GET(기본값)/POST)
>GET 방식: 요청 시 사용자가 입력한 값들이 url 주소에 노출되는 방식
>POST 방식: 요청 시 사용자가 입력한 값들이 url 주소에 노출되지 않는 방식=>주로 민감한 정보를 넘길 때 쓰는 것이 좋음(개인정보)
form 태그 내의 제출(submit) 버튼 클릭 시 action에 지정한 서버로 method에 작성한 요청 전송 방식으로 사용자가 입력했던 값들을 key=value 세트로 전달
=>이 때, 여러 key=value&key=value&... 를 쿼리스트링 이라고 부른다

라디오 버튼과 체크박스 관련 input 태그의 타입

라디오 버튼(type="radio")

-제시한 여러개의 값들 중에서 오로지 한 개의 값만 선택해야 할 때
(name 속성값이 똑같은 것들 끼리 하나의 그룹으로 이어짐)
-제출 시 현재 선택된 (checked) 값이 넘어감

성별:

체크박스(type="checkbox")

-제시한 여러 개의 값들 중에서 여러 개의 값을 선택 가능한 경우
-현재 선택된(checked) 값들이 넘어감
-넘기고자 하는 value 값을 명시해야 함
(나중에 컨트롤러에서 배열 형태로 전달 받을 예정)

select 태그와 option 태그

-사용자에게 직접 키보드로 입력받는게 아니라 내가 제시한 여러 옵셔들 중에서 마우스로 클릭해서 선택할 수 있게끔 해주는 태그
-제출 시 현재 선택된 (selected) 값이 넘어감
-각각의 option에 value 속성값을 넣으면 value 값이 전달됨
-만일, value를 명시 안했을 경우 기본적으로 option 태그의 content 영역(시작태그와 종료태그 사이)의 값이 value로 넘어감
